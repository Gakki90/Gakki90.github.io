linux文件权限

权限分为三个组：[用户][用户组][其他人]
每一个组占三个符号

-rwxr-xr-x 1 root root   207 9月  23 16:44 sh04.sh*
//rwx代表用户可以读，写，执行，r-x代表用户组可以读，执行，r-x代表其他人可以读，执行，如果第一个字符为d那么表明他是一个目录

1.2 shell 重定向：

linux的io输入输出有三种：

  1. Standard Input 代码 0
  2. Standard Output 代码 1
  3. Standard Error 代码 2
    # ls test.sh test1.sh 1>success.txt 2>&1
将错误信息2与标准输出1一起输出到success.txt中
如果是这样
# command 1>file 2>file 2和1会互相覆盖内容
# command 1>file 2>$1 这样就不会覆盖

< 代表输入重定向

输出重定向

代表追加到文件尾，不覆盖

command >/dev/null 2>&1 与command 2>&1 >/dev/null

前者先将标准输出输出到/dev/null中,再将错误输出重定向到标准输出,所以都输出到/dev/null

后者先将错误输出重定向到标准输出也就是输出到屏幕上,再将标准输出到/dev/null,只是把原来的标准输出输出到/de/null中,错误输出重定向到标准输出会显示在屏幕上;

标准输出与错误输出同时输出:command 2>file 1>file 会因抢占file文件的管道二出现乱码/确实/覆盖等现象(即两个输出流操作一个文件的写入),所以要用2>&1 进行重定向到一个输出流才可以全部输出到一个文件

### 1.2 变量的设置规则
1. 两边不能直接空格，要用“或者‘
2. 变量不能以数字开头
3. 用$表示这是个变量,$本身也是一个变量，表示线程代号
4. \表示转义字符
5. 可以用`反引号或者$(命令)表示这是另一个命令。注意反引号和单引号的区别
6. 为变量增加新的内容：PATH=${PATH}:/home/bin
7. export将变量设置为环境变量
8. unset 用于取消变量

Q: 双引号和单引号的区别？
A: 变量的设置中，双引号中可以有变量，而单引号会直接输出语句，不会输出变量，单引号称为强引用，双引号称为弱引用

### 1.3 read，array，declare

read atest

echo $atest//从键盘读取并输出

read -p “please input your name” -t 30 named

echo $named//有提示符，并最多显示30秒

declare [-aixr] variable

-a 声明为数组
-i 声明为数字
-x export，声明为环境变量
-r 只读

var[1]=”string”

echo ${var[1]}//数组的输出用${}

数值的加法要用 sum=$(($x+$y))这种格式

### 1.4 与文件系统和程序的限制关系：ulimit

ulimit [-SHacdfltu]

可以用来限制文件的大小等参数

### 1.5 变量内容的删除替代与替换

${变量#关键字} //从头开始第一个关键字，最短匹配
${变量##关键字} //从头开始第一个关键字，最长匹配
${变量%关键字} //从尾开始第一个关键字，最短匹配
${变量%%关键字} //从尾开始第一个关键字，最长匹配
${变量/旧字符串/新字符串} //替换第一个关键字
${变量//旧字符串/新字符串} //替换所有关键字

alias lm=’ls -l |more’ //设置命令的别名

history //查看历史命令

!! //执行上一次的命令

### 1.7 通配符
符号|意义
---|:--:
* |代表0到无穷多个任意字符
？|代表一个字符，任意的一个
[]|代表括号内的一个字符，[abcd]代表abcd中任意一个
[-]|代表括号中编码顺序内的所有字符，[0-9]
[^]|代表反向选择，[^abc]代表任意一个字符，只要不是abc中的就行
### 1.8 命令执行的判断依据 ；|| &&
上一条命令如果执行成功的话$?就会返回0
1. &&与||：

cmd1 && cmd2 //命令1执行成功那么就执行命令2，失败就不执行命令2

cmd2 || cmd2 //命令1执行不成功就执行命令2，成功就不执行命令2

要注意&&与||连用时，不同的顺序可能导致不同的结果

2. ; :分号代表两个命令无联系，
### 1.9 管道命令
使用|对前面的命令获取的结果进行处理

ll /etc | less //只显示少部分的etc下面的文件

#### 1.9.1 选取命令 cut

echo $PATH | cut -d ‘;’ -f 3,5 //将path中的路径以;分割，输出第三个位置和第五个位置的字符串

echo $PATH | cut -c 12- //输出path路径中从低13个字符以后的字符

last 输出用户的登录记录

last | cut -d ‘ ‘ -f 1 //输出第一列的数据

#### 1.9.2 选取命令 grep
cut 是对找到的字符串进行裁剪，而grep是根据字符串去查找相应的记录

grep [-acinv] [–color=auto] ‘目标字符串’ filename

-v 反向选择
-i 忽略大小写
–color=auto 对目标字符串显示特殊颜色

#### 1.9.3排序命令：sort wc uniq

sort [-fbmnrtuk] [file or stdin] 默认按文字类型排序

uniq [-ic] //删除重复的数据，仅列出一个显示

-i 忽略大小写
-c 统计次数

wc [-lwm] //对文件中行，字数，字符数进行统计

-l 只统计行
-w 字（英文单字）
-m 字符

#### 1.9.4 双向重定向：tee

tee [-a] file //将获取到的数据输入到文件中

-a 以累加（append）方式

last| tee last.list 将last获取到的数据写入last.list中，并且输出到屏幕上

#### 1.9.5 字符转换命令：tr col join paste expand

tr ‘[a-z]’ ‘[A-Z]’ 用大写替换全部小写
tr -d ‘:’ 删除：号

* col将tab键换成空格键
* join处理两个文件的数据
* paste也是处理两个文件间的数据
* expand将tab键转换成空格
#### 1.9.6 切割命令： split

split [-bl] file PREFIX
-b 单位，切成的文件大小
-l 以行数来添加

split -b 300k /etc/termcap termcap

cat termcap* >> termcap 合并前缀为termcap的文件

#### 1.9.7 -
减号- 代表上一次命令的输出作为这一次命令的输入

tar -cvf - /home | tar -xvf -


#### 1.9.8 参数代换： xargs
很多命令不支持管道命令，那么就用xargs代替该命令

find /sbin -perm +7000 |ls -l

ls -l不是管道命令，所以就无法执行

find /sbin -perm +7000 |xargs ls -l


## 2.0 正则表达式的使用
### 2.1 特殊正则式
[:alnym:] 代表英文字母大小写和数字，即0-9，A-Z，a-z
[:alpha:] 代表任何英文字母大小写，即a-z，A-Z
[:digit:] 代表数字，即0-9
### 2.2 高级Grep用法

grep [-A] [-B] [-n] ‘String’ filename //查找String

-A后可接数字，代表after，后续的行
-B后可接数字，代表before，前面的行
-n表示开启行数显示

grep -n ‘the’ 1.txt // 查询包含the的行

grep -vn ‘the’ 1.txt //查询不包含the的行

grep -in ‘the’ 1.txt //查询the的行，部分大小写

正则表示式|代表的意思
:-|:-:|
t[ae]st|代表tast，test
[^g]00|代表不已g开头的oo字符串
^the|代表the开头的行
\.$|代表以.结束的行
^$|代表空行
g..d|代表以g和d中间有两个任意字符
g\*g|代表\*前面的g有0到无穷多个
go\\{2,4\\}g|代表o有2到4个
### 2.3 sed用法
sed以行为单位进行处理，通常用于编辑文件
sed本身也是一个管道命令，对行进行操作

cat /etc/passwd | sed ‘2,5d’ //删除2到5行

cat /etc/passwd | sed -n ‘5,7p’ //只展示5到7行，一定要加-n

sed ‘s/String1/String2/g’ //用字符串2替换字符串1


### 2.4 扩展正则表达式
扩展正则表示式用egrep表示

RE字符|意义与范例
:-:|:-:
+|重复一个或一个以上的前一个RE字符
？|零个或一个的前一个RE字符
\||用or的方式查找字符串
()|查找组字符串：比如查找glad和good，就可以egrep -n 'g(la|oo)d express_regular.txt

### 2.5 awk
数据处理工具，相比sed作用于一整行的处理，awk作用于一整列的处理，用于根据内容分析并处理

last -n 5|awk ‘{printf $1 “\t” $3}’ //输出第一行和第三行

### 3.2 ./ , source ,sh命令的区别
三个命令都可以运行脚本，但./在运行之前需要赋权
不同点在于：./和sh都是在子进程中运行，运行完后销毁进程，所以不保存局部变量，但source是在当前进程中运行，保存变量。


### 3.3 test命令
test 命令用于测试文件是否存在以及他的其他信息

test -e filename //查看文件是否存在

### 3.4 判断符号 []
在判断符号[]中我们可以去判断两数是否相等

[ “$yn” == “Y” -o “$yn” == “y” ] && echo “OK, continue!” && exit 0 

这里由于dash和bash不兼容，bash中默认判断符号为单个=号，运行脚本会出现unexpected operator
1. ：sudo dpkg-reconfigure dash   选NO
将ubuntu 默认的shell 链接的dash 改成传统的 bash \
2. 使用=号

### 3.5 shell script 的默认变量（\$1,$2...)
假如我们在执行脚本的命令后面加上几个变量，那么就可以在脚本中引用

sh sh07.sh one two three
那么$0就代表sh7.sh,$1就代表one，依次$#就代表变量的树木，$@就打印所有的变量

shift用于偏移量

sh sh07.sh one two three
在脚本中写上shift 2 ，那么$0就指向了one
